## 2026-06-11 — Fable 5 (`claude-fable-5`), Claude Code v2.1.170

- **Ingested `addyosmani/web-quality-skills`** (all six skills read in parallel: performance, core-web-vitals, accessibility, seo, best-practices, web-quality-audit; MIT). Treated as authoritative on web-quality substance. Verdict: near-zero overlap with speed-daemon — that repo works at the page/network/main-thread layer, this fleet's gap was exactly there. Nothing contradicts the optimistic-UI default; core-web-vitals' "visual feedback provided immediately" is the same instinct one layer down. Every adoption below was adapted in place; no cross-skill dependencies — the source skills appear only as optional depth references in quality-audit's routing table.
- **speed-daemon — fourth leg: local heavy work.** The network isn't the only render gate; a long main-thread task is the same failure with no spinner to blame. New pattern: cheap state echo → yield (`scheduler.yield()`, `setTimeout(0)` fallback with its priority caveat; `useTransition` in React) → heavy work after the paint → analytics to `requestIdleCallback`. INP gives "feels slow" a number (≤ 200ms at p75; input < 50ms / processing < 100ms / presentation < 50ms); the `web-vitals` attribution build named for field debugging. Pre-ship check and the re-test protocol gain the expensive-input shape (filter/search over a large list). Description and `when_to_use` extended to sluggish-input and INP complaints.
- **quality-audit — two new dimensions plus WCAG 2.2 sharpening.** Web vitals (code level): unsized media, LCP image priority, font-display with metric-matched fallbacks, third-party script strategy and facades, above-viewport injection, virtualization, LCP content in initial HTML — framed as the code shapes that decide the field numbers, since a code audit can't measure p75. Security & best practices: `npm audit` (vulnerable deps = P0), raw HTML sinks, CSP/HSTS/header set, SRI on uncontrolled CDN assets (polyfill.io 2024), secure cookies, hidden source maps, deprecated APIs, passive listeners, global error handlers. Accessibility fallback gains the numbers: 4.5:1 / 3:1 contrast, 24px target floor (SC 2.5.8), focus-not-obscured (SC 2.4.11), single-pointer drag alternative (SC 2.5.7), focus-indicator contrast.
- **design-engineer — citation precision.** Hit-area item now states the real WCAG 2.2 floors (24px AA per SC 2.5.8; 44px AAA per SC 2.5.5; the skill's 40px floor clears AA) — it previously cited SC 2.5.5 for a 40px floor that matches no actual criterion. Scroll-margin item extended from anchored sections to keyboard focus under sticky bars (SC 2.4.11), with the global `:focus { scroll-margin }` rule in `references/polish.md`.
- **Held from the same source:** no SEO skill (the personal-scope geo/seo fleet already owns that ground — triage gap not real) and no standalone accessibility or security skills (their high-leverage content folded into quality-audit's fallback checklists instead).

## 2026-06-10 — Fable 5 (`claude-fable-5`), Claude Code v2.1.170

- **Convention: one-line pins.** Inline `Earned against` comments reduced to a single line across all skills; their history migrated to this file. `bin/preship-check` warns when a pin outgrows one line. Artifacts never narrate the sessions that produced them.
- **All cook/serve frontmatter descriptions rewritten** for listing effectiveness: job + scope + strongest triggers in `description`; `when_to_use` as one compact line (bullet lists render as run-on text in the skill listing). Model-visible total 12,007 → 9,556 chars; no distinct trigger keyword dropped; duplication and body-restatement removed.
- **skill-forge** — "Description first" gains rule 5: the 1,536-char cap is a truncation guard, not a target (job + scope + triggers lands at 600–900 chars); a fleet of near-cap descriptions overflows the shared ~1%-of-context listing budget, evicting least-used descriptions — the silent under-triggering the padding was meant to prevent. Never restate body discipline in a description; for `paths:` skills the body auto-loads when the rules apply. Earned from this fleet's own drift (two skills within 35 chars of truncation).
- **speed-daemon** — gained `paths:` globs; its `when_to_use` claimed auto-load on reactive UI files but no globs existed.
- **Fleet frontmatter pass: request classes over phrase lists.** Triggering is semantic, not lexical — the model reads the listing and matches meaning — so every cook/serve `when_to_use` was rewritten from quoted-phrase lists ("write something on X", "draft a piece about Y") to a statement of the request class plus only the anchors the class doesn't imply (skill names, goal phrasings like "make this less AI-sounding"). Phrase lists sample the center of the request distribution while misses happen at the boundary, and an enumerated list reads as exhaustive — the unlisted case looks out of scope. Same pass: em-dash connectors and cross-skill routing clauses ("composes with", "pairs with", "NOT for X, use Y") dropped from descriptions; the deference notes already live in the bodies. skill-forge "Description first" rule 2 and references/triggering.md now teach the class-first shape; "Distill from source material" gained the blast-radius-of-one rule (ingesting into a skill never adds content or back-references to a neighboring skill).
- **saltintesta / flavored-md — cross-coupling reverted.** A 2026-06-09 ingest had woven the two skills into each other: register/layout paragraphs added to saltintesta's body, flavored-md leaning on saltintesta for its cut-the-fluff half, and (2026-06-10) a rewritten saltintesta description referencing flavored-md. All reverted: saltintesta's original description restored, the register/layout paragraphs removed (that ground lives in flavored-md's "The reader scans" and "Cut the fluff" sections), flavored-md made self-contained with a single optional "if saltintesta is available" pointer. Rule earned: ingesting material into one skill never adds content or references to a neighboring skill; cross-references are soft, optional, and one-directional.
- **harness-audit (cook) — new.** Audits a Claude Code setup end-to-end: inventory the standing surfaces → quantify per-session token cost → four checks (self-consistency, duplication across scopes, enforcement parity, scope discipline) → quantified triage. First worked example: this repo's own setup, which surfaced ~4,600 tokens/session of personal-skill descriptions, the kitchen's 12 skills double-listed (project symlinks + user-scope plugins), a loaded file describing itself as not loaded, and an "always run preship" claim with no enforcement. Sunset: on the next major model release, re-test whether a skill-withheld session inventories scopes, quantifies cost, and checks claims-vs-behavior unaided; if so, delete.
- **Repo machinery** — `.claude/hooks/preship-gate.sh` + committed `.claude/settings.json`: PreToolUse hook blocks `git commit` when `bin/preship-check` fails. `CLAUDE.local.md` slimmed to session-relevant facts; install/troubleshooting playbooks moved to the unloaded `FYI.md`. `bin/preship-check` gains an advisory description-length warning (>1,100 chars, skipping `disable-model-invocation` skills) and the pin-length warning.
- **STATE.md → v2.1.170** — Fable 5 section (opt-in via `/model fable`, `best` alias, effort default `high`, thinking not disableable, classifier fallback to Opus incl. workspace-context triggers, 1M always-on via API, pricing/rollout); In-flux entry for the time-boxed subscription window. **HACKS.md re-pinned to v2.1.170** — Fable classifier-fallback gotcha, `/model best`, `--safe-mode`, `fallbackModel`, `disableBundledSkills`, `/cd`, and the `MAX_THINKING_TOKENS=0` Fable exception.
